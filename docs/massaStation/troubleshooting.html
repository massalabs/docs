Troubleshooting

This page will help you solve some common issues you might encounter while using Massa Station.

Installation‚Äã

On Windows, after installation, my computer lost internet connection‚Äã

Some computers will lose internet connection after the installation. This is a known issue.

To fix this issue, simply restart your computer. If the issue persists, please follow the instructions to get the installer logs and open a new issue on the Massa Station repository so we can help you.

Get installer logs‚Äã

This section will help you get the logs of the installation process, please follow the instructions for your operating system.

Once you have the logs, please open a new issue on the Massa Station repository and attach the logs to your issue so we can help you.

Windows‚Äã

  1. Press Win + R, a small window with a text box should open.
  2. In this window, type %TEMP% and press Enter.
  3. A File Explorer window should appear in a Temp directory.
  4. In this directory, search for the file named MASSASTATION_INSTALLER_{VERSION}.log ( With {VERSION} being the version of Massa Station you are trying to install).

MacOS‚Äã

  1. Do not close the installer window, if you already did, please restart the installation process until it fails and do not close the installer window.
  2. Press CMD + L.
  3. A small window named "Installer Log" should appear at the bottom of your screen.
  4. On the top right of the window, you should see a "Save" button, click on it and save the file in an easy access location, for example your Download directory.

Linux‚Äã

If you used the GUI installation method, please try to install Massa Station using the Terminal method.

If you used the Terminal installation method, please send us the output of the command you used to install Massa Station.

Massa Station‚Äã

Massa Station isn't starting‚Äã

To check what is happening, you can open Massa Station logs. See Get Massa Station logs to know how to get them.

Once you have the logs, please go at the end of the file and look for the last error. If you see an error like this:

listen tcp :80: bind: address already in use

That means that another application is already using the port 80. You can either stop the application using the port 80 or change its port.

I get a page with the error "DNS_PROBE_FINISHED_NXDOMAIN"‚Äã

This error means that your computer can't resolve the domain name of Massa Station. This can happen in multiple cases:

The DNS server isn't installed‚Äã

For Windows users, if you unselected the option to install the DNS server during the installation of Massa Station, .massa domain names can't be resolved.

To fix this, you can either reinstall Massa Station and make sure to select the option to install the DNS server or you can manually change your DNS server to redirect the Massa Station domain name to your computer as explained in the manual installation section.

If the problem persists, please go to the Web browser is using another DNS server section.

The DNS server isn't running‚Äã

If the DNS server is installed but not running, .massa domain names can't be resolved. Please restart your computer to make sure that the DNS server is running.

If the problem persists, please go to the next section.

Your web browser is using another DNS server‚Äã

If the DNS server is installed and running, the problem might be that your web browser is using another DNS server. In Chromium based browsers, you can go to your browser settings and search for "Use secure DNS" and disable it.

If the problem persists, please open an issue with the logs of Massa Station as explained in the Get Massa Station logs section.

I get a page with the error "ERR_CONNECTION_REFUSED"‚Äã

This error means that your computer can't connect to Massa Station. This might be because Massa Station isn't running or because the redirection to station.massa isn't working.

Please check that Massa Station is running and retry to open https://station.massa/ in your web browser.

If you still have the error, please open http://localhost/ in your web browser. If you get the same error, that means that Massa Station isn't running. In that case, please follow the instructions provided in the Massa Station isn't starting section.

Get Massa Station logs‚Äã

In case you encounter an issue that isn't listed in the FAQ or previously on this page, we will need the logs of Massa Station to help you. This section will explain you how to get them.

Please make sure to follow the instructions for your operating system.

Once you have the logs, please open a new issue on the Massa Station repository and attach the logs to your issue so we can help you.

Windows‚Äã

  1. Copy C:\Program Files (x86)\MassaStation\logs\
  2. Press Win + R, a small window with a text box should open.
  3. Paste the previously copied text in the window and press Enter.
  4. A File Explorer window should appear in a the Massa Station logs directory
  5. In this directory, search for the file named massastation.log

MacOS‚Äã

  1. Copy this path: /usr/local/share/massastation/logs/
  2. Open finder
  3. Click on the "Go" button at the top of your screen
  4. Click on the "Go to folder..." button at the bottom of the menu
  5. Paste the path you copied previously
  6. Press Enter

You should be able to see 2 log files:

  • massastation.log
  • massastation-repair.log

Those are the log files you need.

Linux‚Äã

It is located in /usr/local/share/massastation/logs. You can access it using the Files app or with a terminal using cd. Once in the directory, search for the file named massastation.log.

Massa Wallet‚Äã

Get Massa Wallet logs‚Äã

In case you encounter an issue that isn't listed in the FAQ or previously on this page, we will need the logs of Massa Wallet to help you. This section will explain you how to get them.

Please make sure to follow the instructions for your operating system.

Once you have the logs, please open a new issue on the Massa Wallet repository and attach the logs to your issue so we can help you.

Windows‚Äã

  1. Copy %APPDATA%\massa-station-wallet\
  2. Press Win + R, a small window with a text box should open.
  3. In this window, paste what you just copied and press Enter.
  4. A File Explorer window should appear in an APPDATA directory
  5. In this directory, search for the file named station-massa-wallet.log

MacOS‚Äã

  1. Open Finder
  2. On the top right you should see an application bar with different options such as File, Edit and View
  3. Click on the Go and in the dropdown select Library
  4. Look for a directory named Application Support and open it
  5. Open the folder named massa-station-wallet
  6. In this directory, search for the file named station-massa-wallet.log

Linux‚Äã

The log file is located in ~/.config/massa-station-wallet/. You can access it using the Files app or with a terminal using cd. Once in the directory, search for the file named station-massa-wallet.log.

Massa Wallet can't be opened error‚Äã

So far, this issue happens only on Linux because some dependencies aren't installed correctly.

Linux‚Äã

  1. Open your terminal
  2. Verify witch version of this packet you have libwebkit2gtk with apt list --installed libwebkit2gtk
  3. The required version is 0.4-37, so you need to install it with this cmd sudo apt install libwebkit2gtk-4.0-37. If the version installed is already the 0.4-37, please open an issue here.
  4. Once the library installed, restart Massa Wallet module by turning it off and on again.
